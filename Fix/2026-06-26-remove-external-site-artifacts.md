# Remove External Site Artifacts

## User request

Remove remaining unnecessary external-site-derived files and references, make the reference data consistent, and make the site copy say the data was read from game files.

## What changed

- Removed old external-site scraper scripts and npm entries.
- Removed old scrape manifests from `src/data` and `public/data`.
- Removed old downloaded item image PNGs from `public/assets/items`.
- Removed old temporary external-site scrape outputs from `tmp`.
- Removed old Fix records for the previous scraper workflow.
- Updated README and item image folder notes to describe the game-file extraction workflow.
- Updated English and Japanese site copy to state that static JSON is generated by reading permitted local game files.

## Verification performed

- Searched project files for old external-site scraper names and URLs; no matches remained in active project paths.
- Confirmed generated data JSON has no external-site references.
- Confirmed `public/assets/items` no longer contains item PNG files.

## Follow-up notes

- Game-file data extraction remains available through `npm run extract:game-data`.
- Item images still use placeholders unless manually supplied local images are added and the extractor is rerun with `--use-existing-images`.
