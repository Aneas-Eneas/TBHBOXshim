# TBH Chest Simulator

TBH Chest Simulator is an unofficial fan-made Task Bar Hero chest opening simulator built with React, Vite, TypeScript, Tailwind CSS, React Router, and react-i18next.

The Japanese site name is `TBH BOXシミュレーター`.

## Development

Install dependencies:

```bash
npm install
```

Start the local dev server:

```bash
npm run dev
```

The dev server binds to `0.0.0.0`, so phones on the same Wi-Fi can open it with the PC's LAN IP:

```text
http://<PC_LAN_IP>:5173/
```

If the phone cannot connect, confirm the PC and phone are on the same network and allow Node/Vite through Windows Firewall for private networks.

Build for production:

```bash
npm run build
```

The production output is generated in `dist`.

## Cloudflare Pages

Use these Cloudflare Pages settings:

## Extracting Data From Local Game Files

The simulator data is generated by reading the permitted local Task Bar Hero game files. The published site still loads static JSON only; visitors do not trigger scraping or game-file reads.

Install UnityPy once:

```bash
python -m pip install UnityPy -t .unitypy_vendor
```

Then regenerate simulator data:

```bash
npm run update:data
```

By default the extractor reads:

```text
C:\Program Files (x86)\Steam\steamapps\common\TaskbarHero
```

Use a custom install path when needed:

```bash
npm run extract:game-data -- --game-dir "D:\SteamLibrary\steamapps\common\TaskbarHero"
```

Outputs:

- `src/data/items.json`
- `src/data/items.gear.lv80.json`
- `src/data/items.materials.json`
- `src/data/chests.json`
- `src/data/stages.json`
- `src/data/meta.json`
- `src/data/game_extract_manifest.json`
- matching runtime JSON files under `public/data`

The extractor reads the Unity TextAssets for item, box, drop, stage, and localization data. It generates Normal Monster, Stage Boss, and Act Boss box drop groups from game weights and preserves DLC variants.

## Adding Item Images

Place manually supplied item images in:

```text
public/assets/items/
```

Use filenames that match item ids when possible:

```text
public/assets/items/300001.png
```

Generated item records use `imagePath: null` by default, so the app shows a stable no-image placeholder. The game data extractor preserves the game `IconPath` as `sourceIconPath`.

To export optimized shared item sprites from the local game files, run:

```bash
npm run update:data
```

The exported game sprites are stored as unique icon files under `public/assets/items/icons/`. Multiple item ids can point to the same icon when the game uses the same image for different rarities or market variants.

To opt into manually supplied local image files without exporting game sprites, run:

```bash
npm run extract:game-data -- --use-existing-images
```

## Updating Chest Data

Use the game data extractor for current chest and map data.

Run:

```bash
npm run update:data
```

Local data lives in:

- `src/data/chests.json`
- `src/data/stages.json`
- `src/data/items.json`
- `src/data/meta.json`
- `public/data/chests.json`
- `public/data/stages.json`

The app loads chest, stage, and item data from `public/data` at runtime, so large game datasets are not bundled into the main JavaScript file.

The simulator does not read game files or call an external API in production. Treat the JSON files as permission-cleared data generated by reading the local game files.

Chest drop logic:

- `weight` is used first when present.
- `probability` is used when no usable weight is present.
- One `dropGroup` is selected.
- One item is selected from that group.
- Each opened box returns exactly one reward.

After editing JSON, run:

```bash
npm run build
```

Recommended data update flow after a game update:

```bash
npm run update:data
npm run build
```

## Adding Translations

Translations are defined in:


Minimum supported languages:

- English: `en`
- Japanese: `ja`

Add new keys to both languages before using them in components.

## Replacing AdSense Code

`src/components/AdPlaceholder.tsx` is the placeholder component for future ads.

When AdSense is ready:

1. Replace the placeholder markup with the approved AdSense snippet.
2. Keep ads away from the opening buttons.
3. Do not add text that encourages ad clicks.
4. Rebuild with `npm run build`.

AdSense cannot be fully enabled until Google provides the publisher id and ad unit code.

## ads.txt

The placeholder file is:

```text
public/ads.txt
```

Replace the sample comment with the publisher record provided by AdSense, for example:

```text
google.com, pub-0000000000000000, DIRECT, f08c47fec0942fa0
```
